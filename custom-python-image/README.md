# Custom Python Image

This module defines the custom Python auto-instrumentation image used by the PoC and its integration point with the runtime coordinator.

## Purpose

The image is meant to bundle:

- Python runtime dependencies required by instrumented workloads
- OpenTelemetry packages and selected instrumentor packages
- the runtime coordinator package
- container startup behavior that invokes the coordinator before or alongside the user process

## Relationship to the runtime coordinator

A core design choice in this repository is that the image and the coordinator have different responsibilities:

- **image responsibility**: package observability dependencies so they are available at runtime
- **coordinator responsibility**: decide which instrumentors are actually activated for a given process startup

This means the image can remain broad and reusable while the coordinator applies conservative runtime policy based on detection, mode selection, and startup heuristics.

## Current PoC status

The current Dockerfile and entrypoint are still placeholders, but the runtime coordinator code in `runtime-coordinator/` now models the startup behavior this image is expected to host:

- load runtime config
- detect tracing ownership signals
- choose a coordination mode
- build an instrumentation plan
- activate supported instrumentors/wrappers safely
- emit structured diagnostics

## How this can evolve

Beyond the PoC, this image can be extended to:

- include curated sets of instrumentor packages by workload profile
- expose compatibility metadata for the operator to select the right image
- inject config files or environment variables generated by the operator
- include stronger entrypoint logic for bootstrapping user processes and side effects deterministically
- support versioned runtime coordinator releases independent of application code
